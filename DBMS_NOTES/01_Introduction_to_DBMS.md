# 📘 01_Introduction_to_DBMS

---

## 1️⃣ What is Data?

**Data** is raw facts and figures that do not have meaning on their own.
Example:

```
101, Rahul, 85
```

---

## 2️⃣ What is Information?

**Information** is processed data that is meaningful and useful.
Example:

```
Student ID 101, Name Rahul, Marks 85
```

👉 **Interview line:**

> Data becomes information after processing.

---

## 3️⃣ What is a Database?

A **database** is an organized collection of related data stored in a structured way so that it can be **easily accessed, managed, and updated**.

Examples:

* Student database
* Employee database
* Banking database

---

## 4️⃣ What is DBMS?

**DBMS (Database Management System)** is a software that allows users to **store, retrieve, update, and manage data efficiently** in a database.

### Examples of DBMS:

* MySQL
* Oracle
* PostgreSQL
* SQL Server

👉 **Simple definition for interview:**

> DBMS is software that acts as an interface between the user and the database.

---

## 5️⃣ Why DBMS is Needed?

Before DBMS, data was stored in **file systems**, which had many problems.

DBMS is needed to:

* Store large amounts of data
* Avoid data duplication
* Provide data security
* Allow multiple users to work together
* Maintain data consistency

---

## 6️⃣ Main Functions of DBMS

A DBMS performs the following functions:

1. **Data Storage**
2. **Data Retrieval**
3. **Data Update**
4. **Data Security**
5. **Data Backup & Recovery**
6. **Concurrency Control**

---

## 7️⃣ Characteristics of DBMS

Key features of DBMS:

* Data is stored in **tables**
* Reduces **data redundancy**
* Maintains **data integrity**
* Supports **multiple users**
* Provides **authorization and security**
* Ensures **data consistency**

---

## 8️⃣ Advantages of DBMS

✔ Reduced data redundancy
✔ Better data sharing
✔ Improved data security
✔ Easy data access
✔ Data consistency
✔ Backup and recovery support

---

## 9️⃣ Disadvantages of DBMS

✘ High cost
✘ Complex system
✘ Requires skilled staff
✘ Performance overhead for small applications

👉 **Honest interview point:**

> DBMS is powerful but may be overkill for very small applications.

---

## 🔹 Unique & Smart Interview Points

* DBMS separates **logical data** from **physical storage**
* DBMS supports **ACID properties** for reliable transactions
* DBMS allows **data independence**
* DBMS is the backbone of **data analytics and business reporting**

---

## 📌 Real-World Use Case

**Banking System**

* Customer details stored in database
* Multiple users access data simultaneously
* DBMS ensures correct balance using transactions
