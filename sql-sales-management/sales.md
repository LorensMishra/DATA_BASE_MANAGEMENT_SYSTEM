# 📊 SQL Sales Data Management System

---

## 📌 Project Overview

This project demonstrates a complete SQL-based data management system designed for a sales environment. It covers the full workflow from database creation to advanced query analysis.

The goal of this project is to showcase practical SQL skills and a clear understanding of how data is structured, stored, and analyzed in real-world business scenarios.

---

## 🗂️ Project Structure

```
01_database
02_structure_of_tables
03_data_insert_for_sample
04_output_of_table_structure_with_sample
05_basic_queries_questions
06_join_queries
07_advanced_queries
```

---

## 🏗️ Database Design

The project is built using a relational database approach with the following tables:

* Customers
* Products
* Orders
* Order_Items
* Payments
* Suppliers
* Inventory
* Employees
* Sales
* Logs

### Key Features:

* Proper use of Primary Keys and Foreign Keys
* Well-defined relationships between tables
* Logical separation of data (user, admin, system)

---

## 🧾 Data Insertion

Sample data is inserted to simulate real business scenarios:

* Customers from different regions
* Products with categories and pricing
* Orders with multiple statuses
* Payment records with different methods

This helps in performing meaningful analysis.

## 🔍 Data Verification

All tables are verified using SELECT queries to ensure:

* Data consistency
* Proper relationships
* Correct structure

## 📘 SQL Query Implementation

### 🔹 Basic Queries

* Data retrieval using SELECT
* Filtering using WHERE
* Sorting using ORDER BY
* Aggregations like COUNT, SUM, AVG

---

### 🔹 Join Queries

* INNER JOIN
* LEFT JOIN
* RIGHT JOIN
* Multi-table joins

Used to combine data across multiple tables for analysis.

---

### 🔹 Advanced Queries

* Subqueries
* CASE statements
* HAVING clause
* Window functions

Used to generate business insights such as:

* Top customers
* Sales trends
* Product performance
